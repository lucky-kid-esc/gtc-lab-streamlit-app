**Aim :** To implement the basic graphs such as Path graph, Cycle graph, Complete graph and Complete Bi-pertite graph

### **Theory:** The structural properties of discrete relational systems are formally characterized within the framework of graph theory. A graph G is defined as an ordered triple [![][image1]](https://www.codecogs.com/eqnedit.php?latex=\(V\(G\)%2C%20E\(G\)%2C%20%5Cpsi_G\)#2), where [![][image2]](https://www.codecogs.com/eqnedit.php?latex=V\(G\)#2) is a nonempty set of vertices, [![][image3]](https://www.codecogs.com/eqnedit.php?latex=E\(G\)#2) is a set of edges disjoint from [![][image4]](https://www.codecogs.com/eqnedit.php?latex=V\(G\)#2), and [![][image5]](https://www.codecogs.com/eqnedit.php?latex=%5Cpsi_G#2) is an incidence function that associates each edge with an unordered pair of vertices. In the context of simple graphs, the incidence function [![][image6]](https://www.codecogs.com/eqnedit.php?latex=%5Cpsi_G#2) is often suppressed, such that each edge is represented directly as a subset of [![][image7]](https://www.codecogs.com/eqnedit.php?latex=V\(G\)#2) of cardinality two. These structures facilitate the modeling of mutual relationships, where an edge [![][image8]](https://www.codecogs.com/eqnedit.php?latex=uv%20%5Cin%20E\(G\)#2) indicates adjacency between vertices u and v. The degree of a vertex, denoted [![][image9]](https://www.codecogs.com/eqnedit.php?latex=d_G\(v\)#2), is the number of edges incident with v.

### The classification of simple graphs includes several fundamental families defined by their adjacency patterns. A path graph [![][image10]](https://www.codecogs.com/eqnedit.php?latex=P_n#2) is a connected graph of order n whose vertices can be arranged in a linear sequence [![][image11]](https://www.codecogs.com/eqnedit.php?latex=v_1%2C%20v_2%2C%20%5Cdots%2C%20v_n#2) such that the edge set is [![][image12]](https://www.codecogs.com/eqnedit.php?latex=E\(P_n\)%20%3D%20%5C%7Bv_i%20v_%7Bi%2B1%7D%20%3A%201%20%5Cle%20i%20%3C%20n%5C%7D#2). This configuration implies that [![][image13]](https://www.codecogs.com/eqnedit.php?latex=d\(v_1\)%20%3D%20d\(v_n\)%20%3D%201#2) and [![][image14]](https://www.codecogs.com/eqnedit.php?latex=d\(v_i\)%20%3D%202#2) for [![][image15]](https://www.codecogs.com/eqnedit.php?latex=1%20%3C%20i%20%3C%20n#2). Building upon this, a cycle graph [![][image16]](https://www.codecogs.com/eqnedit.php?latex=C_n#2) for [![][image17]](https://www.codecogs.com/eqnedit.php?latex=n%20%5Cge%203#2) is obtained by the addition of the edge [![][image18]](https://www.codecogs.com/eqnedit.php?latex=v_n%20v_1#2) to [![][image19]](https://www.codecogs.com/eqnedit.php?latex=P_n#2), resulting in a 2-regular connected graph. Maximizing connectivity leads to the complete graph [![][image20]](https://www.codecogs.com/eqnedit.php?latex=K_n#2), wherein every distinct pair of vertices is joined by a unique edge, satisfying [![][image21]](https://www.codecogs.com/eqnedit.php?latex=d\(v\)%20%3D%20n-1#2) for all [![][image22]](https://www.codecogs.com/eqnedit.php?latex=v%20%5Cin%20V\(K_n\)#2). Furthermore, a graph G is bipartite if [![][image23]](https://www.codecogs.com/eqnedit.php?latex=V\(G\)#2) can be partitioned into two disjoint independent sets X and Y; it is a complete bipartite graph [![][image24]](https://www.codecogs.com/eqnedit.php?latex=K_%7Bm%2Cn%7D#2) if every vertex in X is adjacent to every vertex in Y, where [![][image25]](https://www.codecogs.com/eqnedit.php?latex=%7CX%7C%3Dm#2) and [![][image26]](https://www.codecogs.com/eqnedit.php?latex=%7CY%7C%3Dn#2).

### The algorithmic generation of these structures involves specific iterative procedures. To construct [![][image27]](https://www.codecogs.com/eqnedit.php?latex=P_n#2), an algorithm initializes [![][image28]](https://www.codecogs.com/eqnedit.php?latex=V%20%3D%20%5C%7B1%2C%20%5Cdots%2C%20n%5C%7D#2) and performs a single-pass iteration to define [![][image29]](https://www.codecogs.com/eqnedit.php?latex=E%20%3D%20%5C%7B\(i%2C%20i%2B1\)%20%3A%201%20%5Cle%20i%20%3C%20n%5C%7D#2). For [![][image30]](https://www.codecogs.com/eqnedit.php?latex=C_n#2), the same procedure is followed with the terminal step of establishing the edge [![][image31]](https://www.codecogs.com/eqnedit.php?latex=\(n%2C%201\)#2). The construction of [![][image32]](https://www.codecogs.com/eqnedit.php?latex=K_n#2) requires a combinatorial approach, typically implemented via nested iterations to ensure every [![][image33]](https://www.codecogs.com/eqnedit.php?latex=u%2C%20v%20%5Cin%20V#2) with [![][image34]](https://www.codecogs.com/eqnedit.php?latex=u%20%5Cneq%20v#2) are assigned an edge, resulting in [![][image35]](https://www.codecogs.com/eqnedit.php?latex=%7CE%7C%20%3D%20%5Cbinom%7Bn%7D%7B2%7D#2). The [![][image36]](https://www.codecogs.com/eqnedit.php?latex=K_%7Bm%2Cn%7D#2) structure is generated by a Cartesian-product-style iteration where edges are established for every [![][image37]](https://www.codecogs.com/eqnedit.php?latex=\(u%2C%20v\)#2) such that [![][image38]](https://www.codecogs.com/eqnedit.php?latex=u%20%5Cin%20X#2) and [![][image39]](https://www.codecogs.com/eqnedit.php?latex=v%20%5Cin%20Y#2).

#### NETWORKX FUNCTIONS USED

The script utilizes the NetworkX library for the mathematical construction of the graphs.

nx.Graph(): This constructor initializes an empty graph object that creates an undirected graph by default. It serves as the container for the nodes and edges.

add\_nodes\_from(range(1,6)): This efficient helper function adds multiple nodes at once. The "range(1,6)" argument generates the numbers 1, 2, 3, 4, and 5 automatically.

add\_edge(u, v): This is the core function for defining structure. It establishes a link between node u and node v. If the nodes do not already exist, this function automatically creates them.

nx.spring\_layout(p5): This is a force-directed algorithm. It simulates the graph as a physical system where edges are springs and nodes are electrically charged particles that repel each other. It calculates the optimal (x, y) coordinates to display the graph so that it isn't cluttered.

nx.draw(): This is the primary rendering command.

ax: Tells the function which specific subplot to draw on.pos: Takes a dictionary of coordinates (either manually defined or calculated by spring\_layout) to fix the nodes in space.with\_labels=True: Ensures the integer ID of the node (1, 2, etc.) is printed on top of the node circle.

#### MATPLOTLIB FUNCTIONS USED

Matplotlib is used here solely for creating the canvas and organizing the four graphs into a grid.

plt.subplots(2,2): This function is a convenient way to generate a figure and a set of subplots simultaneously.

The argument (2,2) specifies a grid with 2 rows and 2 columns.it returns "fig" (the overall window) and "ax" (a numpy array containing the 4 subplot axes).The code accesses these axes using array indexing, such as ax\[0,0\] for top-left, ax\[0,1\] for top-right, ax\[1,0\] for bottom-left, and ax\[1,1\] for bottom-right.

set\_title(): This method assigns a specific text header to each individual subplot, helping the viewer identify which graph theory concept is being displayed (e.g., "K5" vs "P5").

plt.show(): This function finishes the script execution by opening an interactive window that displays the final rendered figure containing all four graphs.

\# graphs\_in.py (with networkx functions)  
import matplotlib.pyplot as plt  
import networkx as nx  
fig, ax \= plt.subplots(2, 2\)  
def set\_border(axis, color, width):  
    axis.set\_axis\_on()  
    axis.tick\_params(left=False, bottom=False, labelleft=False, labelbottom=False)  
    for spine in axis.spines.values():  
        spine.set\_edgecolor(color)  
        spine.set\_linewidth(width)  
        spine.set\_visible(True)  
\# P5 \- Path Graph  
p5 \= nx.path\_graph(range(1, 6))  
pos\_p5 \= nx.spring\_layout(p5)  
nx.draw(  
    p5,  
    ax=ax\[0, 0\],  
    pos=pos\_p5,  
    with\_labels=True,  
    node\_color="orange",  
    edge\_color="red"  
)  
ax\[0, 0\].set\_title("Path Graph (P5)")  
set\_border(ax\[0, 0\], "red", 1.2)  
\# K23 \- Complete Bipartite Graph  
k23 \= nx.complete\_bipartite\_graph(\[1, 2\], \[3, 4, 5\])  
pos\_k23 \= {1: \[1, 2\], 2: \[3, 2\], 3: \[0, 0\], 4: \[2, 0\], 5: \[4, 0\]}  
nx.draw(  
    k23,  
    ax=ax\[0, 1\],  
    pos=pos\_k23,  
    with\_labels=True,  
    node\_color="cyan",  
    edge\_color="blue"  
)  
ax\[0, 1\].set\_title("Complete Bipartite Graph (K23)")  
set\_border(ax\[0, 1\], "blue", 1.2)  
\# C5 \- Cyclic Graph  
c5 \= nx.cycle\_graph(range(1, 6))  
pos\_c5 \= nx.circular\_layout(c5)  
nx.draw(  
    c5,  
    ax=ax\[1, 0\],  
    pos=pos\_c5,  
    with\_labels=True,  
    node\_color="lime",  
    edge\_color="green"  
)  
ax\[1, 0\].set\_title("Cyclic Graph (C5)")  
set\_border(ax\[1, 0\], "green", 1.2)  
\# K5 \- Complete Graph  
k5 \= nx.complete\_graph(range(1, 6))  
pos\_k5 \= nx.circular\_layout(k5)  
nx.draw(  
    k5,  
    ax=ax\[1, 1\],  
    pos=pos\_k5,  
    with\_labels=True,  
    node\_color="violet",  
    edge\_color="purple"  
)  
ax\[1, 1\].set\_title("Complete Graph (K5)")  
set\_border(ax\[1, 1\], "purple", 1.2)  
plt.tight\_layout()  
plt.show()

\# graph\_mn.py (without networkx functions)  
import matplotlib.pyplot as plt  
import networkx as nx  
fig, ax \= plt.subplots(2,2)  
def set\_border(axis, color , width):  
    axis.set\_axis\_on()  
    axis.tick\_params(left=False, bottom=False, labelleft=False, labelbottom=False)  
    for spine in axis.spines.values():  
        spine.set\_edgecolor(color)  
        spine.set\_linewidth(width)  
        spine.set\_visible(True)  
\# P5  
p5 \= nx.Graph()  
p5.add\_nodes\_from(range(1,6))  
for i in range(1,5):  
    p5.add\_edge(i,i+1)  
pos\_p5 \= nx.spring\_layout(p5)  
nx.draw(  
        p5,  
        ax \= ax\[0,0\],  
        pos \= pos\_p5,  
        with\_labels=True,  
        node\_color="orange",  
        edge\_color="red"  
        )  
ax\[0,0\].set\_title("Path Graph (P5)")  
set\_border(ax\[0,0\], "red", 1.2)  
\# K23  
k23 \= nx.Graph()  
k23.add\_nodes\_from(range(1,6))  
for i in \[1,2\]:  
    for j in \[3,4,5\]:  
        k23.add\_edge(i,j)  
pos\_k23 \= {1:\[1,2\], 2:\[3,2\], 3:\[0,0\], 4:\[2,0\], 5:\[4,0\]}  
nx.draw(  
        k23,  
        ax \= ax\[0,1\],  
        pos \= pos\_k23,  
        with\_labels=True,  
        node\_color="cyan",  
        edge\_color="blue"  
        )  
ax\[0,1\].set\_title("Complete Bipartite Graph (K23)")  
set\_border(ax\[0,1\], "blue", 1.2)  
\# C5  
c5 \= nx.Graph()  
c5.add\_nodes\_from(range(1,6))  
for i in range(1,5):  
    c5.add\_edge(i, i+1)  
c5.add\_edge(5,1)  
pos\_c5 \= nx.circular\_layout(c5);  
nx.draw(  
        c5,  
        ax \= ax\[1,0\],  
        pos \= pos\_c5,  
        with\_labels=True,  
        node\_color="lime",  
        edge\_color="green"  
        )  
ax\[1,0\].set\_title("Cyclic Graph (C5)")  
set\_border(ax\[1,0\], "green", 1.2)  
\# K5  
k5 \= nx.Graph()  
k5.add\_nodes\_from(range(1,6))  
for i in range (1,6):  
    for j in \[n for n in range(1,6) if n\!=i\]:  
        k5.add\_edge(i,j)  
pos\_k5 \= nx.circular\_layout(k5)  
nx.draw(  
        k5,  
        ax \= ax\[1,1\],  
        pos \= pos\_k5,  
        with\_labels=True,  
        node\_color="violet",  
        edge\_color="purple"  
        )  
ax\[1,1\].set\_title("Complete Graph (K5)")  
set\_border(ax\[1,1\], "purple", 1.2)  
plt.tight\_layout()  
plt.show()
